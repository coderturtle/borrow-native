## Security-Conscious Reviewer — Borrow Native Design Critique

**Scope:** narrow check for unsafe-practice modeling risk only. Findings below, most important first.

**1. Test-gaming risk (terminal-velocity's Module 04 finding) is acknowledged in principle but not closed as a gap.** The gremlin-design doc states plainly: "Coachgremlin's two-tier grading extension is designed but not yet run against a real learner attempt in this or any Rust context — same 'designed, not yet evidenced' status `terminal-velocity`'s own agent-native-interaction plan was left in" (workshop-gremlin-design.md). That's honest, but neither doc specifies a concrete mitigation for the *exact* failure mode named in this review's prompt: a learner's agent editing the test file itself to force green, rather than fixing the code. The design says the deterministic tier is "cargo test green... either passes or it doesn't — no judgment call" (workshop-design.md), but nowhere states that fixture/test files are protected, hashed, or diffed for tampering before Coachgremlin's conceptual pass runs. Given this is a self-paced, facilitator-free public repo (no one watching the harness live), this is exactly the setup where that gaming vector recurs. Recommend the design explicitly add: test/fixture integrity check (e.g., diff against canonical test file) as a precondition before the conceptual tier even engages.

**2. The `unsafe`-as-escape-hatch risk is well-named, but only at the general two-tier-grading level, not per-module.** The design correctly flags it: Coachgremlin checks "did the learner reach for `.clone()`/`unsafe`/`Rc<RefCell<_>>` to silence the compiler rather than understanding what it was telling them?" — a strong, specific warning. But it lives only in the general grading-philosophy section; Module 04 (Generics/Traits/Lifetimes) and Module 06 (Concurrency) — the two modules where compiler fights are hardest and `unsafe`/lifetime-extension tricks are the most tempting escape — don't repeat or reinforce this warning in their own takeaway rows. Since exercise specifics are explicitly deferred to Coachgremlin build-time, this isn't a hard defect, but the design pass had the chance to hard-wire the warning into those two specific modules and didn't.

**3. Module 06 (Fearless Concurrency) itself models nothing unsafe.** Takeaway is a "channel vs. shared-state decision guide," anchored to Book ch16, which is Rust's safe-concurrency-primitives chapter. No finding here — this module's framing is sound.

No other safety-modeling concerns found.
