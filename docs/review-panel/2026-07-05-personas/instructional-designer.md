# Instructional Designer — Critique of Borrow Native Design

**1. The capstone's stated deliverable isn't a coding exercise, so its deterministic gate is undefined.**
Every other module pairs a coding exercise (deterministically gated per the design's own core claim) with a takeaway artifact. Module 07's row in "What you keep" states only: *"A personal Rust diagnostic playbook compressing the whole arc into one checklist."* Nothing in either table describes a program the learner builds. If the capstone's primary artifact is a checklist, what does `cargo test`/`cargo clippy` gate? This directly fails the workshop's own headline principle ("deterministic tier — primary... conceptual tier — secondary") for the one module that most needs to prove it isn't just Module 06 with more steps. As written, "compressing the arc into a checklist" is indistinguishable from a documentation exercise, not a synthesis exercise — the exact failure mode this lens exists to catch.

**2. Module 04's hard prerequisite on Module 03 is asserted, not argued — the one gap in an otherwise-justified graph.**
Rows 03, 05, and 06 each carry parenthetical reasoning for their prerequisites, and the "Why this order" prose walks through each in turn (borrowing needs ownership; structs need `&self`/`&mut self`; error handling needs enums + generics; concurrency needs ownership/borrowing + `Send`/`Sync`). Row 04 (`Generics, Traits & Lifetimes`) lists prerequisite "03" with no parenthetical, and the prose's discussion of module 04 — *"Generics/traits/lifetimes stay one module because they're mutually entangled..."* — justifies why the three concepts aren't split, never why they depend on structs/enums/pattern matching at all. The Book's own canonical trait-bound example the doc cites (`fn largest<T: PartialOrd>(list: &[T]) -> &T`) needs no struct. This is the single unreconciled edge in the dependency table.

**3. No per-module gate is actually specified — only asserted once, globally.**
The two-tier gate is stated workshop-wide, but the arc table has no "Gate" column parallel to its "Takeaway" column, so there's nothing yet to check module-by-module whether Coachgremlin is being asked to grade something clippy already catches. Forgivable pre-content, but worth requiring before authoring starts.
