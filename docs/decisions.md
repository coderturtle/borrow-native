# Decisions: Borrow Native

## ADR Log

| Date | Decision | Rationale |
|---|---|---|
| 2026-07-05 | Initial scaffold as factory-output | Learn Rust the agent-native way: agent-literate practitioners work through harness-driven exercises with the compiler and test suite as a deterministic gate, and Coachgremlin grading the conceptual/idiomatic layer on top. |
| 2026-07-05 | Naming pass complete: final name is **Borrow Native** (slug `borrow-native`) | Chosen by coderturtle from naming-agent candidates (Borrow Native, Fearless Loop, Rust, Verified, Compile or Concede) — all four confirmed available under `github.com/coderturtle` before presenting. Echoes Terminal Velocity's "X Native" naming pattern (agent-*native*) fused with Rust's defining concept (the *borrow* checker), signaling this is the second workshop in the same family rather than an unrelated one-off. GitHub repo renamed (`gh repo rename`, old URL redirects), local dir/git remote/repo-local mind-palace mirror renamed, vault card moved (explicit human authorisation given for this specific vault mutation, matching Terminal Velocity's precedent), `.hekton/project.yaml` and all current-state docs updated in place via find-replace across `*.md`/`*.yaml`. |
| 2026-07-05 | Second real run of the Workshop Gremlin (`~/hekton/gremlins/workshop/workshop-gremlin.md`), first on a subject other than agentic engineering | Deliberately chosen as this Gremlin's next run per its own Follow-Up Actions ("watched across 2-3 more workshops before either claims v1 stability") and `terminal-velocity`'s note that this Gremlin had only run once, entirely on its own founding subject. Surfaced a genuine augmentation before any content was built: Rust has an objective checker (compiler/`cargo test`/`cargo clippy`) `terminal-velocity`'s subject never had, making a **deterministic gate tier** a strict upgrade over Coachgremlin's purely subjective rubric — written back into `workshop-gremlin.md` (Design Principle 2 extension + new "Variant: Tech/Language Workshops" section), `coachgremlin.md` (two-tier grading in Workflow step 3), and `workshop-review-panel.md` (Instructional Designer persona check), each as a minor version step, not a new roster item. |
| 2026-07-05 | 6-module + capstone arc, anchored to the Rust Book/Rustlings/Exercism (all fetched directly, not recalled from memory) | Rustlings' exercise-to-chapter mapping and Exercism's `config.json` prerequisite arrays gave real, checkable evidence for a concept-dependency graph rather than an invented sequence. Iterators/Closures folded into Module 04 as an optional extension rather than a standalone module, reusing `terminal-velocity`'s validated "required core + optional extensions" shape. |
| 2026-07-05 | Ran the Workshop Review Panel (7 personas) against the naming + design docs before building the module skeleton | Matches `terminal-velocity`'s validated timing (cheapest to fix while still prose). Caught two real citation errors in the curriculum-anchor research (Rustlings exercise count; Exercism concept-tag-vs-exercise-name conflation) and a structural gap (the capstone's gate was undefined). All cheap fixes applied same-session; full report and triage in `docs/review-panel/2026-07-05-initial-design.md`. |
| 2026-07-05 | Built the module skeleton, brand layer, README rework, and Astro-on-Pages site, deliberately without a custom domain | Reused `terminal-velocity`'s Astro site verbatim where generic (layouts, styles, content-collection loader), adapted only naming/copy and the `base` path (`/borrow-native/`, standard GitHub Pages project hosting). A custom domain, like `terminal-velocity.coderturtle.io`'s, needs a real Route53/Terraform change via `agentic-infra-lab` — explicitly out of scope for this scaffolding session; can be revisited once the workshop has real content. Locally validated: `npm run build` and `astro check` both clean, base-path links confirmed in built HTML. `npm audit` found 4 inherited vulnerabilities from the starter (same ones `terminal-velocity` found and left untriaged) — logged as RISK-0002, not yet triaged, must be addressed before the first real deploy. |
| 2026-07-05 | Corrected the certification-companion target from "the Rust Foundation certification" to Ardan Labs' Rust certification | Researched at coderturtle's request, against six external links (Rust Foundation Trusted Training's own page, a New Stack article on its launch, and four Rust-learning opinion pieces). Found the Rust Foundation's Trusted Training (RFTT) program accredits *training providers* (five founding orgs: Mainmatter, Integer 32, Wyliodrin, Doulos, Ferrous Systems), not individual learners — no exam, no individual credential exists there. Ardan Labs' Rust certification (proctored, 100 MCQ, 90 min, 80% to pass, $99, unlimited retakes, 2-year validity) is a real, individually-achievable equivalent, chosen by coderturtle over Linux Foundation's LFD480 (a $3,495 attendance-only course with no pass/fail exam, weaker fit for "dogfood to get certified"). Full reasoning: `docs/workshop-design.md`'s "External Validation" section. |
| 2026-07-05 | Added Module 07 (Async Programming); capstone renumbered 07 → 08 | Checking this arc against Ardan's published exam topics (memory safety, type systems, async, testing/debugging practice) found the one gap the original Book/Rustlings/Exercism curriculum-anchor research hadn't caught: async wasn't covered anywhere in the original 6-module arc. Positioned after Module 06 (Concurrency), matching the Rust Book's own ch16-then-ch17 sequencing; depends on Module 04 (`Future` is a trait) and Module 06 (easiest to tell apart from threads once threads are fluent). Takeaway: an async-vs-threads decision guide, paired with Module 06's as a matched set. |
| 2026-07-05 | coderturtle committed to personally sitting the real Ardan exam once module content exists | This workshop's own dogfooding evidence, mirroring `terminal-velocity`'s Coachgremlin dry-run discipline (a real attempt, evidenced, not just designed) — gives Borrow Native a falsifiable external claim `terminal-velocity` had no equivalent way to make. Recorded as a stated intent in `docs/next-actions.md` and `docs/workshop-design.md`, explicitly not yet evidenced. |
