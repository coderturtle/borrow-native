| 2026-07-05 | Corrected the certification-companion target from "the Rust Foundation certification" to Ardan Labs' Rust certification | Researched at coderturtle's request, against six external links (Rust Foundation Trusted Training's own page, a New Stack article on its launch, and four Rust-learning opinion pieces). Found the Rust Foundation's Trusted Training (RFTT) program accredits *training providers* (five founding orgs: Mainmatter, Integer 32, Wyliodrin, Doulos, Ferrous Systems), not individual learners — no exam, no individual credential exists there. Ardan Labs' Rust certification (proctored, 100 MCQ, 90 min, 80% to pass, $99, unlimited retakes, 2-year validity) is a real, individually-achievable equivalent, chosen by coderturtle over Linux Foundation's LFD480 (a $3,495 attendance-only course with no pass/fail exam, weaker fit for "dogfood to get certified"). Full reasoning: `docs/workshop-design.md`'s "External Validation" section. |
| 2026-07-05 | Added Module 07 (Async Programming); capstone renumbered 07 → 08 | Checking this arc against Ardan's published exam topics (memory safety, type systems, async, testing/debugging practice) found the one gap the original Book/Rustlings/Exercism curriculum-anchor research hadn't caught: async wasn't covered anywhere in the original 6-module arc. Positioned after Module 06 (Concurrency), matching the Rust Book's own ch16-then-ch17 sequencing; depends on Module 04 (`Future` is a trait) and Module 06 (easiest to tell apart from threads once threads are fluent). Takeaway: an async-vs-threads decision guide, paired with Module 06's as a matched set. |
| 2026-07-05 | coderturtle committed to personally sitting the real Ardan exam once module content exists | This workshop's own dogfooding evidence, mirroring `terminal-velocity`'s Coachgremlin dry-run discipline (a real attempt, evidenced, not just designed) — gives Borrow Native a falsifiable external claim `terminal-velocity` had no equivalent way to make. | 2026-07-05 | Built Module 01's real exercise (`merge_customer_totals`) and ran Coachgremlin's first real dry run for this workshop: a correct move-based attempt and a deliberately naive, honest (non-adversarial) clone-heavy attempt | **Load-bearing finding, not just a clean pass:** both attempts pass `cargo test` (6/6) and default `cargo clippy -- -D warnings` (zero output) identically — the deterministic gate, as this module defines it, cannot distinguish them at all. Only Coachgremlin's conceptual tier catches the unnecessary clone. Checked (not assumed) whether a stricter lint level closes the gap: `clippy::pedantic` does flag the naive attempt, but via a different, more general lint (`needless_pass_by_value`) than the clone itself, bundled with an unrelated stylistic nit, and `pedantic` isn't what most real Rust projects gate on by default — decided to keep the module's stated gate at default clippy rather than quietly upgrading it to make this one gap disappear. Full grading: `runs/2026-07-05-module-01-dry-run/grading.md` and `retro.md`. | 2026-07-05 | Pivoted from eight independent per-module toy fixtures to one shared throughline project, `fixtures/relay/` (a restartable-handoff CLI for hybrid human-agent team pacing), retiring the standalone `merge_customer_totals` exercise | coderturtle's direction: modules should build one real, usable tool, not eight disconnected checklists. Reuses `terminal-velocity`'s validated "one shared fixture" pattern, shaped differently on purpose: `receipts` is one static function engineered five ways (agentic-engineering practice is five lenses on one problem); `relay` grows one real feature per module (Rust's concept arc is an additive dependency chain, not five lenses). Product is a direct implementation of the "Restartable Handoff Loop" pattern named in `terminal-velocity`'s own research — pomodoro-style checkpoints, each producing a goal/diff/evidence/risks handoff summary and notifying a human. Deliberately narrower than `agent-mission-control-lab`'s existing multi-agent cockpit, to avoid competing reimplementation. Tagline settled as "hybrid human-agent team pacing," not the burnout/drug-adjacent "Claude crack" framing floated earlier (real brand risk for a public repo). Working name "relay" (handoff/baton-pass metaphor), not a formal naming ceremony, revisitable later. Full module-to-feature mapping and reasoning: `docs/workshop-design.md`'s "The shared project: relay" section and `fixtures/relay/SPEC.md`. |
