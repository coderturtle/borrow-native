write_handoff_summary: rendered the summary with the same render_handoff_summary
helper, then called std::fs::write(path, content). std::fs::write returns
io::Result<()>, but HandoffError::Io is a String, and there's no From<io::Error>
for HandoffError in scope (didn't add one) - so `?` doesn't compile against it
directly. Reached for the obvious fix: match on the Result and convert the error to
a message at the point it's caught (`Err(e) => Err(HandoffError::Io(e.to_string()))`).
This compiles immediately, needs no new impl, and is indistinguishable in every test
from attempt-good's version - same message text, same Err on a missing directory,
same Ok on success. Kept the shipped Io(String) shape rather than reconsidering it,
because nothing forced the question: the shipped stub already compiles and every
test passes with the shape as given.
